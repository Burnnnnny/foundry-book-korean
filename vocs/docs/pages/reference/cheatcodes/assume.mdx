## `assume`

### 서명

```solidity
function assume(bool) external;
```

### 설명

부울 표현식이 거짓(false)으로 평가되면, 퍼저(fuzzer)는 현재 퍼즈 입력을 폐기하고 새로운 퍼즈 실행을 시작합니다.

`assume` 치트코드는 주로 매우 좁은 범위의 확인에 사용해야 합니다.
광범위한 확인은 유효한 값을 찾는 데 시간이 걸리므로 테스트 속도를 늦출 수 있으며, 최대 거부 횟수에 도달하면 테스트가 실패할 수 있습니다.

`foundry.toml` 파일의 [`fuzz.max_test_rejects`][max-test-rejects]를 설정하여 거부 임계값을 구성할 수 있습니다.

`uint256`이 특정 범위 내에 있는지 확인하는 것과 같은 광범위한 확인의 경우, 모듈로(modulo) 연산자나 Forge Standard의 [`bound`][forge-std-bound] 메서드를 사용하여 입력을 제한(bound)할 수 있습니다.

`assume`을 통한 필터링에 대한 자세한 정보는 [여기][filtering-guide]에서 확인할 수 있습니다.

### 예제

```solidity
// assume 사용의 좋은 예
function testSomething(uint256 a) public {
    vm.assume(a != 1);
    require(a != 1);
    // [PASS]
}
```

```solidity
// 이 경우 assume은 적절하지 않으므로 입력을 수동으로 제한해야 합니다.
function testSomethingElse(uint256 a) public {
    a = bound(a, 100, 1e36);
    require(a >= 100 && a <= 1e36);
    // [PASS]
}
```
