## `assumeNoRevert`

### 서명

```solidity
function assumeNoRevert() external;
```

### 설명

다음 호출이 리버트(revert)되면 퍼저(fuzzer)는 현재 퍼즈 입력을 폐기하고 새로운 퍼즈 실행을 시작합니다.

최대 거부 횟수에 도달하면 테스트가 실패할 수 있습니다.

`foundry.toml` 파일의 [`fuzz.max_test_rejects`][max-test-rejects]를 설정하여 거부 임계값을 구성할 수 있습니다.

### 예제

특정 범위의 금액을 요구하는 함수의 경우:
```solidity
function doSomething(uint256 amount) public {
    require(amount > 100 ether && amount < 1_000 ether);
}
```
리버트는 폐기되어 테스트가 통과합니다 (또는 최대 거부 횟수에 도달하면 실패):
```solidity
function testSomething(uint256 amount) public {
    vm.assumeNoRevert();
    target.doSomething(amount);
    // [PASS]
}
```
