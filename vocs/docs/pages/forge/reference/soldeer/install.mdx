## forge soldeer install

종속성을 설치합니다.

인수와 함께 사용하면 종속성이 구성(config)에 추가됩니다. 인수 없이 사용하면 누락된 모든 종속성을 설치합니다.

Examples:
- 전체 설치: soldeer install
- 레지스트리에서 추가: soldeer install lib_name~2.3.0
- 사용자 정의 URL로 추가: soldeer install lib_name~2.3.0 --url
https://foo.bar/lib.zip
- Git으로 추가: soldeer install lib_name~2.3.0 --git git@github.com:foo/bar.git
- Git으로 추가 (커밋): soldeer install lib_name~2.3.0 --git
git@github.com:foo/bar.git --rev 05f218fb6617932e56bf5388c3b389c3028a7b73
- Git으로 추가 (태그): soldeer install lib_name~2.3.0 --git
git@github.com:foo/bar.git --tag v2.3.0
- Git으로 추가 (브랜치): soldeer install lib_name~2.3.0 --git
git@github.com:foo/bar.git --branch feature/baz

```bash
$ forge soldeer install --help
```

```txt
Usage: forge soldeer install [OPTIONS] [DEPENDENCY~VERSION]

Arguments:
  [DEPENDENCY~VERSION]
          종속성 이름과 버전. 물결표(tilde)로 구분됩니다. 버전은 항상 필수입니다.
          
          없는 경우, 이 명령은 누락된 모든 종속성을 설치합니다.

Options:
      --url <ZIP_URL>
          종속성 zip 파일의 URL.
          
          Example: https://my-domain/dep.zip

      --git <GIT_URL>
          종속성 저장소의 URL.
          
          Example: git@github.com:foo/bar.git

      --rev <REV>
          Git 커밋 해시

      --tag <TAG>
          Git 태그

      --branch <BRANCH>
          Git 브랜치

  -g, --regenerate-remappings
          설정되면 기존 리매핑을 삭제하고 다시 생성합니다.

  -d, --recursive-deps
          설정되면 종속성을 재귀적으로 설치합니다 (git 서브모듈 또는 soldeer를 통해)

      --clean
          모든 종속성을 재설치하여 클린 설치 수행

      --config-location <CONFIG_LOCATION>
          묻지 않고 구성(config) 위치 지정.
          
          자동 감지로 구성 위치를 결정할 수 없는 경우 사용자에게 묻는 메시지를
          표시하지 않습니다.
          
          [possible values: foundry, soldeer]

  -h, --help
          도움말 인쇄 ('-h'로 요약 보기)

  -j, --threads <THREADS>
          사용할 스레드 수. 0으로 지정하면 기본값은 논리 코어 수
          
          [aliases: --jobs]

Display options:
      --color <COLOR>
          로그 메시지의 색상

          가능한 값:
          - auto:   색상 출력 사용 여부를 지능적으로 추측 (기본값)
          - always: 색상 출력 강제
          - never:  색상 출력 강제 비활성화

      --json
          로그 메시지를 JSON으로 포맷팅

      --md
          로그 메시지를 Markdown으로 포맷팅

  -q, --quiet
          로그 메시지를 인쇄하지 않음

  -v, --verbosity...
          로그 메시지의 상세 수준.
          
          상세 수준을 높이려면 여러 번 전달 (예: -v, -vv, -vvv).
          
          문맥에 따라 상세 수준은 다른 의미를 가짐.
          
          예를 들어, EVM의 상세 수준은 다음과 같음:
          - 2 (-vv): 모든 테스트에 대한 로그 인쇄.
          - 3 (-vvv): 실패한 테스트에 대한 실행 트레이스 인쇄.
          - 4 (-vvvv): 모든 테스트에 대한 실행 트레이스 및 실패한 테스트에 대한 설정(setup) 트레이스 인쇄.
          - 5 (-vvvvv): 스토리지 변경 및 줄 번호가 포함된 백트레이스를 포함하여
            모든 테스트에 대한 실행 및 설정 트레이스 인쇄.

For more information, read the README.md
```
