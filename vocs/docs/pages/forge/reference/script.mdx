## forge script

스마트 컨트랙트를 스크립트로 실행하여 온체인으로 보낼 수 있는 트랜잭션을 구축합니다.

```bash
$ forge script --help
```

```txt
Usage: forge script [OPTIONS] <PATH> [ARGS]...

Arguments:
  <PATH>
          실행하려는 컨트랙트. 파일 경로 또는 컨트랙트 이름.
          
          동일한 파일에 여러 컨트랙트가 존재하는 경우 --target-contract로
          대상 컨트랙트를 지정해야 합니다.

  [ARGS]...
          스크립트 함수에 전달할 인수

Options:
  -j, --threads <THREADS>
          사용할 스레드 수. 0으로 지정하면 기본값은 논리 코어 수
          
          [aliases: --jobs]

      --target-contract <CONTRACT_NAME>
          실행하려는 컨트랙트의 이름
          
          [aliases: --tc]

  -s, --sig <SIG>
          컨트랙트에서 호출하려는 함수의 서명 또는 원시 calldata
          
          [default: run]

      --priority-gas-price <PRICE>
          EIP1559 트랜잭션에 대한 가스당 최대 우선 수수료
          
          [env: ETH_PRIORITY_GAS_PRICE=]

      --legacy
          EIP1559 트랜잭션 대신 레거시 트랜잭션 사용.
          
          EIP1559가 없는 일반적인 네트워크에 대해 자동으로 활성화됩니다.

      --broadcast
          트랜잭션 브로드캐스트

      --batch-size <BATCH_SIZE>
          트랜잭션의 배치 크기.
          
          배치를 사용할 수 없거나 `--slow`가 활성화된 경우 이 값은 무시되고
          1로 설정됩니다.
          
          [default: 100]

      --skip-simulation
          온체인 시뮬레이션 건너뛰기

  -g, --gas-estimate-multiplier <GAS_ESTIMATE_MULTIPLIER>
          가스 추정치에 곱할 상대적 백분율
          
          [default: 130]

      --unlocked
          `--sender` 인수를 발신자로 사용하여 `eth_sendTransaction`을 통해 전송

      --resume
          이전에 실패하거나 타임아웃된 트랜잭션 제출을 재개합니다.
          
          스크립트를 다시 시뮬레이션하지 않으며, 논스(nonce)가 동일하게 유지된
          것으로 가정합니다.
          
          예: 트랜잭션 N의 논스가 22인 경우 계정의 논스도 22여야 하며,
          그렇지 않으면 실패합니다.

      --multi
          이 옵션이 있으면 --resume 또는 --verify는 다중 체인 배포로 가정됩니다.

      --debug
          디버거에서 스크립트 열기.
          
          broadcast보다 우선순위가 높습니다.

      --dump <PATH>
          모든 디버거 단계를 파일로 덤프

      --slow
          이전 트랜잭션이 확인되고 성공한 후에만 트랜잭션이 전송되도록 합니다.

      --non-interactive
          큰 컨트랙트를 배포할 때 나타날 수 있는 대화형 프롬프트 비활성화.
          
          컨트랙트 크기 제한에 대한 자세한 정보는 EIP-170 참조:
          [https://eips.ethereum.org/EIPS/eip-170](https://eips.ethereum.org/EIPS/eip-170)

      --disable-code-size-limit
          스크립트 실행 중 컨트랙트 크기 제한 비활성화

      --disable-labels
          트레이스에서 레이블 비활성화

      --etherscan-api-key <KEY>
          Etherscan (또는 동등한) API 키
          
          [env: ETHERSCAN_API_KEY=]

      --verify
          스크립트 영수증에서 발견된 모든 컨트랙트 검증 (있는 경우)

      --with-gas-price <PRICE>
          레거시 트랜잭션의 가스 가격, 또는 EIP1559 트랜잭션의 가스당 최대
          수수료. wei 단위 또는 단위 유형이 포함된 문자열로 지정.
          
          예: 1ether, 10gwei, 0.01ether
          
          [env: ETH_GAS_PRICE=]

      --timeout <TIMEOUT>
          트랜잭션 브로드캐스팅에 사용할 타임아웃
          
          [env: ETH_TIMEOUT=]

  -h, --help
          도움말 인쇄 ('-h'로 요약 보기)

Display options:
  -v, --verbosity...
          로그 메시지의 상세 수준.
          
          상세 수준을 높이려면 여러 번 전달 (예: -v, -vv, -vvv).
          
          문맥에 따라 상세 수준은 다른 의미를 가짐.
          
          예를 들어, EVM의 상세 수준은 다음과 같음:
          - 2 (-vv): 모든 테스트에 대한 로그 인쇄.
          - 3 (-vvv): 실패한 테스트에 대한 실행 트레이스 인쇄.
          - 4 (-vvvv): 모든 테스트에 대한 실행 트레이스 및 실패한 테스트에 대한 설정(setup) 트레이스 인쇄.
          - 5 (-vvvvv): 스토리지 변경 및 줄 번호가 포함된 백트레이스를 포함하여
            모든 테스트에 대한 실행 및 설정 트레이스 인쇄.

  -q, --quiet
          로그 메시지를 인쇄하지 않음

      --json
          로그 메시지를 JSON으로 포맷팅

      --md
          로그 메시지를 Markdown으로 포맷팅

      --color <COLOR>
          로그 메시지의 색상

          가능한 값:
          - auto:   색상 출력 사용 여부를 지능적으로 추측 (기본값)
          - always: 색상 출력 강제
          - never:  색상 출력 강제 비활성화

Cache options:
      --force
          캐시 및 아티팩트 폴더를 지우고 재컴파일

Build options:
      --no-cache
          캐시 비활성화

      --dynamic-test-linking
          동적 테스트 링킹 활성화

      --skip <SKIP>...
          이름에 주어진 필터가 포함된 파일 빌드 건너뛰기.
          
          `test`와 `script`는 `.t.sol`과 `.s.sol`의 별칭입니다.

Linker options:
      --libraries <LIBRARIES>
          사전 링크된 라이브러리 설정
          
          [env: DAPP_LIBRARIES=]

Compiler options:
      --ignored-error-codes <ERROR_CODES>
          오류 코드로 solc 경고 무시

  -D, --deny <LEVEL>
          지정된 진단 수준에서 컴파일러 오류가 트리거됩니다.
          
          사용 중단된 `--deny-warnings` 플래그를 대체합니다.
          
          가능한 값: - `never`: 어떤 진단도 오류로 처리하지 않음. -
          `warnings`: 경고를 오류로 처리. - `notes`: 경고와 노트(notes)
          모두 오류로 처리.

          가능한 값:
          - never:    항상 0 코드로 종료
          - warnings: 경고가 발견되면 0이 아닌 코드로 종료
          - notes:    노트나 경고가 발견되면 0이 아닌 코드로 종료

      --no-auto-detect
          `solc` 버전을 자동 감지하지 않음

      --use <SOLC_VERSION>
          빌드할 solc 버전을 지정하거나 로컬 solc 경로 지정.
          
          유효한 값은 `x.y.z`, `solc:x.y.z` 또는 `path/to/solc` 형식입니다.

      --offline
          네트워크에 액세스하지 않음.
          
          누락된 solc 버전은 설치되지 않음.

      --via-ir
          Yul 중간 표현(intermediate representation) 컴파일 파이프라인 사용

      --use-literal-content
          컴파일 시 URL 대신 리터럴 콘텐츠만 사용하도록 변경

      --no-metadata
          바이트코드에 메타데이터를 추가하지 않음.
          
          `bytecode_hash`를 `none`으로, `cbor_metadata`를 `false`로
          설정하는 것과 동일함.

      --ast
          컴파일러 출력에 AST를 JSON으로 포함

      --evm-version <VERSION>
          대상 EVM 버전

      --optimize [<OPTIMIZE>]
          Solidity 최적화기(optimizer) 활성화
          
          [possible values: true, false]

      --optimizer-runs <RUNS>
          실행 횟수(runs)는 배포된 코드의 각 오퍼코드가 컨트랙트 수명 동안
          대략 얼마나 자주 실행될지를 지정합니다. 이는 코드 크기(배포 비용)와
          코드 실행 비용(배포 후 비용) 간의 균형 매개변수입니다. `optimizer_runs`
          매개변수가 `1`이면 짧지만 비싼 코드를 생성합니다. 반면, 더 큰
          `optimizer_runs` 매개변수는 더 길지만 가스 효율적인 코드를 생성합니다.

      --extra-output <SELECTOR>...
          컨트랙트 아티팩트에 포함할 추가 출력.
          
          예제 키: evm.assembly, ewasm, ir, irOptimized, metadata
          
          전체 설명은 다음을 참조하세요:
          [https://docs.soliditylang.org/en/v0.8.13/using-the-compiler.html#input-description](https://docs.soliditylang.org/en/v0.8.13/using-the-compiler.html#input-description)

      --extra-output-files <SELECTOR>...
          별도 파일에 기록할 추가 출력.
          
          유효한 값: metadata, ir, irOptimized, ewasm, evm.assembly

Project options:
  -o, --out <PATH>
          컨트랙트 아티팩트 폴더 경로

      --revert-strings <REVERT>
          리버트 문자열 구성.
          
          가능한 값: "default", "strip" (제거), "debug" (Solidity 생성 리버트
          문자열) 및 "verboseDebug"

      --build-info
          빌드 정보 파일 생성

      --build-info-path <PATH>
          빌드 정보 파일이 기록될 디렉토리의 출력 경로

      --root <PATH>
          프로젝트의 루트 경로.
          
          기본적으로 Git 저장소의 루트(있는 경우) 또는 현재 작업 디렉토리입니다.

  -C, --contracts <PATH>
          컨트랙트 소스 디렉토리

  -R, --remappings <REMAPPINGS>
          프로젝트의 리매핑

      --remappings-env <ENV>
          환경(변수)에서 가져온 프로젝트 리매핑

      --cache-path <PATH>
          컴파일러 캐시 경로

      --lib-paths <PATH>
          라이브러리 폴더 경로

      --hardhat
          Hardhat 스타일 프로젝트 레이아웃 사용.
          
          다음과 동일함: `--contracts contracts --lib-paths node_modules`.
          
          [aliases: --hh]

      --config-path <FILE>
          구성(config) 파일 경로

Wallet options - raw:
      --interactives <NUM>
          개인 키를 입력하기 위한 대화형 프롬프트 열기.
          
          입력할 키의 개수 값을 받습니다.
          
          [default: 0]

  -i, --interactive
          개인 키를 입력하기 위한 대화형 프롬프트 열기

      --private-keys <RAW_PRIVATE_KEYS>
          제공된 개인 키 사용

      --private-key <RAW_PRIVATE_KEY>
          제공된 개인 키 사용

      --mnemonics <MNEMONICS>
          지정된 경로의 니모닉 파일에 있는 니모닉 문구 사용

      --mnemonic-passphrases <PASSPHRASE>
          니모닉에 BIP39 암호문(passphrase) 사용

      --mnemonic-derivation-paths <PATH>
          지갑 파생 경로.
          
          --mnemonic-path 및 하드웨어 지갑 모두에서 작동합니다.

      --mnemonic-indexes <INDEXES>
          주어진 니모닉 인덱스의 개인 키 사용.
          
          --mnemonics, --ledger, --aws 및 --trezor와 함께 사용할 수
          있습니다.
          
          [default: 0]

Wallet options - keystore:
      --keystore <PATHS>
          주어진 폴더의 파일 이름을 사용하여 키스토어 사용
          
          [env: ETH_KEYSTORE=]
          [aliases: --keystores]

      --account <ACCOUNT_NAMES>
          파일 이름을 사용하여 기본 키스토어 폴더(~/.foundry/keystores)의
          키스토어 사용
          
          [env: ETH_KEYSTORE_ACCOUNT=]
          [aliases: --accounts]

      --password <PASSWORDS>
          키스토어 비밀번호.
          
          --keystore와 함께 사용됩니다.

      --password-file <PATHS>
          키스토어 비밀번호 파일 경로.
          
          --keystore와 함께 사용됩니다.
          
          [env: ETH_PASSWORD=]

Wallet options - hardware wallet:
  -l, --ledger
          Ledger 하드웨어 지갑 사용

  -t, --trezor
          Trezor 하드웨어 지갑 사용

Wallet options - remote:
      --aws
          AWS Key Management Service 사용.
          
          AWS_KMS_KEY_IDS (콤마로 구분) 또는 AWS_KMS_KEY_ID 환경 변수 중 하나가
          설정되어 있는지 확인하세요.

      --gcp
          Google Cloud Key Management Service 사용.
          
          GCP_PROJECT_ID, GCP_LOCATION, GCP_KEY_RING, GCP_KEY_NAME,
          GCP_KEY_VERSION 환경 변수가 설정되어 있는지 확인하세요.
          
          참조: [https://cloud.google.com/kms/docs](https://cloud.google.com/kms/docs)

      --turnkey
          Turnkey 사용.
          
          TURNKEY_API_PRIVATE_KEY, TURNKEY_ORGANIZATION_ID, TURNKEY_ADDRESS
          환경 변수가 설정되어 있는지 확인하세요.
          
          참조: [https://docs.turnkey.com/getting-started/quickstart](https://docs.turnkey.com/getting-started/quickstart)

EVM options:
  -f, --fork-url <URL>
          빈 상태에서 시작하는 대신 원격 엔드포인트에서 상태 가져오기.
          
          특정 블록 번호에서 상태를 가져오려면 --fork-block-number 참조.
          
          [aliases: --rpc-url]

      --fork-block-number <BLOCK>
          원격 엔드포인트에서 특정 블록 번호의 상태 가져오기.
          
          --fork-url 참조.

      --fork-retries <RETRIES>
          재시도 횟수.
          
          --fork-url 참조.

      --fork-retry-backoff <BACKOFF>
          오류 발생 시 초기 재시도 백오프(backoff).
          
          --fork-url 참조.

      --no-storage-caching
          RPC 캐싱 사용을 명시적으로 비활성화.
          
          모든 스토리지 슬롯을 엔드포인트에서 완전히 읽음.
          
          이 플래그는 프로젝트의 구성 파일을 덮어씀.
          
          --fork-url 참조.

      --initial-balance <BALANCE>
          배포된 테스트 컨트랙트의 초기 잔액

      --sender <ADDRESS>
          테스트/스크립트를 실행할 주소

      --ffi
          FFI 치트코드 활성화

      --always-use-create-2-factory
          테스트 및 비-브로드캐스팅 스크립트를 포함한 모든 경우에 create 2
          팩토리 사용

      --create2-deployer <ADDRESS>
          사용할 CREATE2 배포자 주소. 구성(config)에 있는 주소를 덮어씀

Fork config:
      --compute-units-per-second <CUPS>
          이 공급자에 대해 가정된 초당 사용 가능한 컴퓨팅 유닛 수 설정
          
          기본값: 330
          
          --fork-url 및
          [https://docs.alchemy.com/reference/compute-units#what-are-cups-compute-units-per-second](https://docs.alchemy.com/reference/compute-units#what-are-cups-compute-units-per-second) 참조

      --no-rpc-rate-limit
          이 노드 공급자에 대한 속도 제한 비활성화.
          
          --fork-url 및
          [https://docs.alchemy.com/reference/compute-units#what-are-cups-compute-units-per-second](https://docs.alchemy.com/reference/compute-units#what-are-cups-compute-units-per-second) 참조
          
          [aliases: --no-rate-limit]

Executor environment config:
      --code-size-limit <CODE_SIZE>
          EIP-170: 바이트 단위의 컨트랙트 코드 크기 제한. 테스트 때문에
          늘리는 데 유용함. 기본값은 0x6000 (~25kb)

      --chain <CHAIN>
          체인 이름 또는 EIP-155 체인 ID
          
          [aliases: --chain-id]

      --gas-price <GAS_PRICE>
          가스 가격

      --block-base-fee-per-gas <FEE>
          블록의 기본 수수료(base fee)
          
          [aliases: --base-fee]

      --tx-origin <ADDRESS>
          트랜잭션 원점(origin)

      --block-coinbase <ADDRESS>
          블록의 코인베이스

      --block-timestamp <TIMESTAMP>
          블록의 타임스탬프

      --block-number <BLOCK>
          블록 번호

      --block-difficulty <DIFFICULTY>
          블록 난이도

      --block-prevrandao <PREVRANDAO>
          블록 prevrandao 값. 참고: 병합(merge) 전에는 이 필드가 mix_hash였음

      --block-gas-limit <BLOCK_GAS_LIMIT>
          블록 가스 제한
          
          [aliases: --gas-limit]

      --memory-limit <MEMORY_LIMIT>
          바이트 단위의 EVM 실행 당 메모리 제한. 이 제한을 초과하면
          `MemoryLimitOOG` 결과가 발생함.
          
          기본값은 128MiB.

      --disable-block-gas-limit
          블록 가스 제한 검사 비활성화 여부
          
          [aliases: --no-block-gas-limit, --no-gas-limit]

      --enable-tx-gas-limit
          Osaka (EIP-7825)에 의해 부과된 트랜잭션 가스 제한 검사 활성화 여부
          
          [aliases: --tx-gas-limit]

      --isolate
          호출 격리 활성화 여부. 격리 모드에서는 모든 최상위 호출이 별도의
          EVM 컨텍스트에서 별도의 트랜잭션으로 실행되어 더 정확한 가스 회계
          및 트랜잭션 상태 변경을 가능하게 함

      --retries <RETRIES>
          검증 재시도 횟수
          
          [default: 5]

      --delay <DELAY>
          검증 시도 사이에 적용할 선택적 지연 시간(초)
          
          [default: 5]

Verifier options:
      --verifier <VERIFIER>
          사용할 컨트랙트 검증 공급자

          가능한 값:
          - etherscan
          - sourcify
          - blockscout
          - oklink
          - custom:     사용자 정의 검증 공급자, Etherscan API와 호환 필요
          
          [default: sourcify]

      --verifier-api-key <VERIFIER_API_KEY>
          사용자 정의 공급자를 사용하는 경우 검증자 API 키
          
          [env: VERIFIER_API_KEY=]

      --verifier-url <VERIFIER_URL>
          사용자 정의 공급자를 사용하는 경우 검증자 URL
          
          [env: VERIFIER_URL=]
```
