## forge create

스마트 컨트랙트를 배포합니다.

```bash
$ forge create --help
```

```txt
Usage: forge create [OPTIONS] <CONTRACT>

Arguments:
  <CONTRACT>
          `<path>:<contractname>` 형태의 컨트랙트 식별자

Options:
      --constructor-args <ARGS>...
          생성자 인수

      --constructor-args-path <PATH>
          생성자 인수를 포함하는 파일 경로

      --broadcast
          트랜잭션 브로드캐스트

      --verify
          생성 후 컨트랙트 검증

      --unlocked
          `--from` 인수 또는 `$ETH_FROM`을 발신자로 사용하여 `eth_sendTransaction`을 통해 전송

      --show-standard-json-input
          `--verify`가 제공된 경우 표준 json 컴파일러 입력을 인쇄합니다.
          
          표준 json 컴파일러 입력은 브라우저에서 수동으로 컨트랙트 검증을
          제출하는 데 사용할 수 있습니다.

      --timeout <TIMEOUT>
          트랜잭션 브로드캐스팅에 사용할 타임아웃
          
          [env: ETH_TIMEOUT=]

  -h, --help
          도움말 인쇄 ('-h'로 요약 보기)

  -j, --threads <THREADS>
          사용할 스레드 수. 0으로 지정하면 기본값은 논리 코어 수
          
          [aliases: --jobs]

Cache options:
      --force
          캐시 및 아티팩트 폴더를 지우고 재컴파일

Build options:
      --no-cache
          캐시 비활성화

      --dynamic-test-linking
          동적 테스트 링킹 활성화

      --skip <SKIP>...
          이름에 주어진 필터가 포함된 파일 빌드 건너뛰기.
          
          `test`와 `script`는 `.t.sol`과 `.s.sol`의 별칭입니다.

Linker options:
      --libraries <LIBRARIES>
          사전 링크된 라이브러리 설정
          
          [env: DAPP_LIBRARIES=]

Compiler options:
      --ignored-error-codes <ERROR_CODES>
          오류 코드로 solc 경고 무시

  -D, --deny <LEVEL>
          지정된 진단 수준에서 컴파일러 오류가 트리거됩니다.
          
          사용 중단된 `--deny-warnings` 플래그를 대체합니다.
          
          가능한 값: - `never`: 어떤 진단도 오류로 처리하지 않음. -
          `warnings`: 경고를 오류로 처리. - `notes`: 경고와 노트(notes)
          모두 오류로 처리.

          가능한 값:
          - never:    항상 0 코드로 종료
          - warnings: 경고가 발견되면 0이 아닌 코드로 종료
          - notes:    노트나 경고가 발견되면 0이 아닌 코드로 종료

      --no-auto-detect
          `solc` 버전을 자동 감지하지 않음

      --use <SOLC_VERSION>
          빌드할 solc 버전을 지정하거나 로컬 solc 경로 지정.
          
          유효한 값은 `x.y.z`, `solc:x.y.z` 또는 `path/to/solc` 형식입니다.

      --offline
          네트워크에 액세스하지 않음.
          
          누락된 solc 버전은 설치되지 않음.

      --via-ir
          Yul 중간 표현(intermediate representation) 컴파일 파이프라인 사용

      --use-literal-content
          컴파일 시 URL 대신 리터럴 콘텐츠만 사용하도록 변경

      --no-metadata
          바이트코드에 메타데이터를 추가하지 않음.
          
          `bytecode_hash`를 `none`으로, `cbor_metadata`를 `false`로
          설정하는 것과 동일함.

      --ast
          컴파일러 출력에 AST를 JSON으로 포함

      --evm-version <VERSION>
          대상 EVM 버전

      --optimize [<OPTIMIZE>]
          Solidity 최적화기(optimizer) 활성화
          
          [possible values: true, false]

      --optimizer-runs <RUNS>
          실행 횟수(runs)는 배포된 코드의 각 오퍼코드가 컨트랙트 수명 동안
          대략 얼마나 자주 실행될지를 지정합니다. 이는 코드 크기(배포 비용)와
          코드 실행 비용(배포 후 비용) 간의 균형 매개변수입니다. `optimizer_runs`
          매개변수가 `1`이면 짧지만 비싼 코드를 생성합니다. 반면, 더 큰
          `optimizer_runs` 매개변수는 더 길지만 가스 효율적인 코드를 생성합니다.

      --extra-output <SELECTOR>...
          컨트랙트 아티팩트에 포함할 추가 출력.
          
          예제 키: evm.assembly, ewasm, ir, irOptimized, metadata
          
          전체 설명은 다음을 참조하세요:
          [https://docs.soliditylang.org/en/v0.8.13/using-the-compiler.html#input-description](https://docs.soliditylang.org/en/v0.8.13/using-the-compiler.html#input-description)

      --extra-output-files <SELECTOR>...
          별도 파일에 기록할 추가 출력.
          
          유효한 값: metadata, ir, irOptimized, ewasm, evm.assembly

Project options:
  -o, --out <PATH>
          컨트랙트 아티팩트 폴더 경로

      --revert-strings <REVERT>
          리버트 문자열 구성.
          
          가능한 값: "default", "strip" (제거), "debug" (Solidity 생성 리버트
          문자열) 및 "verboseDebug"

      --build-info
          빌드 정보 파일 생성

      --build-info-path <PATH>
          빌드 정보 파일이 기록될 디렉토리의 출력 경로

      --root <PATH>
          프로젝트의 루트 경로.
          
          기본적으로 Git 저장소의 루트(있는 경우) 또는 현재 작업 디렉토리입니다.

  -C, --contracts <PATH>
          컨트랙트 소스 디렉토리

  -R, --remappings <REMAPPINGS>
          프로젝트의 리매핑

      --remappings-env <ENV>
          환경(변수)에서 가져온 프로젝트 리매핑

      --cache-path <PATH>
          컴파일러 캐시 경로

      --lib-paths <PATH>
          라이브러리 폴더 경로

      --hardhat
          Hardhat 스타일 프로젝트 레이아웃 사용.
          
          다음과 동일함: `--contracts contracts --lib-paths node_modules`.
          
          [aliases: --hh]

      --config-path <FILE>
          구성(config) 파일 경로

Transaction options:
      --gas-limit <GAS_LIMIT>
          트랜잭션에 대한 가스 제한
          
          [env: ETH_GAS_LIMIT=]

      --gas-price <PRICE>
          레거시 트랜잭션의 가스 가격, 또는 EIP1559 트랜잭션의 가스당 최대
          수수료. wei 단위 또는 단위 유형이 포함된 문자열로 지정.
          
          예: 1ether, 10gwei, 0.01ether
          
          [env: ETH_GAS_PRICE=]

      --priority-gas-price <PRICE>
          EIP1559 트랜잭션에 대한 가스당 최대 우선 수수료
          
          [env: ETH_PRIORITY_GAS_PRICE=]

      --value <VALUE>
          트랜잭션에 보낼 이더(Ether). wei 단위 또는 단위 유형이 포함된
          문자열로 지정.
          
          예: 1ether, 10gwei, 0.01ether

      --nonce <NONCE>
          트랜잭션에 대한 논스(Nonce)

      --legacy
          EIP1559 트랜잭션 대신 레거시 트랜잭션 전송.
          
          EIP1559가 없는 일반적인 네트워크에 대해 자동으로 활성화됩니다.

      --blob
          EIP-4844 블롭 트랜잭션 전송

      --blob-gas-price <BLOB_PRICE>
          EIP-4844 블롭 트랜잭션에 대한 가스 가격
          
          [env: ETH_BLOB_GAS_PRICE=]

      --auth <AUTH>
          EIP-7702 권한 부여 목록.
          
          16진수로 인코딩된 서명된 권한 부여 또는 주소일 수 있습니다.

      --access-list [<ACCESS_LIST>]
          EIP-2930 액세스 목록.
          
          JSON으로 인코딩된 액세스 목록을 받거나 빈 값을 사용하여
          `eth_createAccessList` RPC 호출을 통해 액세스 목록을 생성합니다.
          액세스 목록 부분만 검색하려면 `cast access-list` 명령을 사용하세요.

Rpc options:
  -r, --rpc-url <URL>
          RPC 엔드포인트, 기본값은 http://localhost:8545
          
          [env: ETH_RPC_URL=]

  -k, --insecure
          안전하지 않은 RPC 연결 허용 (유효하지 않은 HTTPS 인증서 수락).
          
          공급자의 내부 런타임 전송 변형이 HTTP인 경우 reqwest 클라이언트가
          유효하지 않은 인증서를 수락하도록 구성합니다.

      --flashbots
          빠른 모드로 Flashbots RPC URL 사용
          ([https://rpc.flashbots.net/fast](https://rpc.flashbots.net/fast)).
          
          이는 등록된 모든 빌더와 트랜잭션을 비공개로 공유합니다.
          
          참조:
          [https://docs.flashbots.net/flashbots-protect/quick-start#faster-transactions](https://docs.flashbots.net/flashbots-protect/quick-start#faster-transactions)

      --jwt-secret <JWT_SECRET>
          RPC 엔드포인트에 대한 JWT 비밀.
          
          JWT 비밀은 RPC에 대한 JWT를 생성하는 데 사용됩니다. 예를 들어,
          다음은 CL `engine_forkchoiceUpdated` 호출을 시뮬레이션하는 데 사용할
          수 있습니다:
          
          cast rpc --jwt-secret <JWT_SECRET> engine_forkchoiceUpdatedV2
          '["0x6bb38c26db65749ab6e472080a3d20a2f35776494e72016d1e339593f21c59bc",
          "0x6bb38c26db65749ab6e472080a3d20a2f35776494e72016d1e339593f21c59bc",
          "0x6bb38c26db65749ab6e472080a3d20a2f35776494e72016d1e339593f21c59bc"]'
          
          [env: ETH_RPC_JWT_SECRET=]

      --rpc-timeout <RPC_TIMEOUT>
          RPC 요청에 대한 타임아웃(초).
          
          지정된 타임아웃은 RPC 요청에 대한 기본 타임아웃을 재정의하는 데
          사용됩니다.
          
          기본값: 45
          
          [env: ETH_RPC_TIMEOUT=]

      --rpc-headers <RPC_HEADERS>
          RPC 요청에 대한 사용자 정의 헤더 지정
          
          [env: ETH_RPC_HEADERS=]

  -e, --etherscan-api-key <KEY>
          Etherscan (또는 동등한) API 키
          
          [env: ETHERSCAN_API_KEY=]

  -c, --chain <CHAIN>
          체인 이름 또는 EIP-155 체인 ID
          
          [env: CHAIN=]

Wallet options - raw:
  -f, --from <ADDRESS>
          발신자 계정
          
          [env: ETH_FROM=]

  -i, --interactive
          개인 키를 입력하기 위한 대화형 프롬프트 열기

      --private-key <RAW_PRIVATE_KEY>
          제공된 개인 키 사용

      --mnemonic <MNEMONIC>
          지정된 경로의 니모닉 파일에 있는 니모닉 문구 사용

      --mnemonic-passphrase <PASSPHRASE>
          니모닉에 BIP39 암호문(passphrase) 사용

      --mnemonic-derivation-path <PATH>
          지갑 파생 경로.
          
          --mnemonic-path 및 하드웨어 지갑 모두에서 작동합니다.

      --mnemonic-index <INDEX>
          주어진 니모닉 인덱스의 개인 키 사용.
          
          --mnemonic-path와 함께 사용됩니다.
          
          [default: 0]

      --retries <RETRIES>
          검증 재시도 횟수
          
          [default: 5]

      --delay <DELAY>
          검증 시도 사이에 적용할 선택적 지연 시간(초)
          
          [default: 5]

Wallet options - keystore:
      --keystore <PATH>
          주어진 폴더 또는 파일에서 키스토어 사용
          
          [env: ETH_KEYSTORE=]

      --account <ACCOUNT_NAME>
          파일 이름을 사용하여 기본 키스토어 폴더(~/.foundry/keystores)의
          키스토어 사용
          
          [env: ETH_KEYSTORE_ACCOUNT=]

      --password <PASSWORD>
          키스토어 비밀번호.
          
          --keystore와 함께 사용됩니다.

      --password-file <PASSWORD_FILE>
          키스토어 비밀번호 파일 경로.
          
          --keystore와 함께 사용됩니다.
          
          [env: ETH_PASSWORD=]

Wallet options - hardware wallet:
  -l, --ledger
          Ledger 하드웨어 지갑 사용

  -t, --trezor
          Trezor 하드웨어 지갑 사용

Wallet options - remote:
      --aws
          AWS Key Management Service 사용.
          
          AWS_KMS_KEY_ID 환경 변수가 설정되어 있는지 확인하세요.

      --gcp
          Google Cloud Key Management Service 사용.
          
          GCP_PROJECT_ID, GCP_LOCATION, GCP_KEY_RING, GCP_KEY_NAME,
          GCP_KEY_VERSION 환경 변수가 설정되어 있는지 확인하세요.
          
          참조: [https://cloud.google.com/kms/docs](https://cloud.google.com/kms/docs)

      --turnkey
          Turnkey 사용.
          
          TURNKEY_API_PRIVATE_KEY, TURNKEY_ORGANIZATION_ID, TURNKEY_ADDRESS
          환경 변수가 설정되어 있는지 확인하세요.
          
          참조: [https://docs.turnkey.com/getting-started/quickstart](https://docs.turnkey.com/getting-started/quickstart)

Wallet options - browser:
      --browser
          브라우저 지갑 사용

      --browser-port <PORT>
          브라우저 지갑 서버 포트
          
          [default: 9545]

      --browser-disable-open
          지갑 연결을 위해 브라우저를 열지 여부

Verifier options:
      --verifier <VERIFIER>
          사용할 컨트랙트 검증 공급자

          가능한 값:
          - etherscan
          - sourcify
          - blockscout
          - oklink
          - custom:     사용자 정의 검증 공급자, Etherscan API와 호환 필요
          
          [default: sourcify]

      --verifier-api-key <VERIFIER_API_KEY>
          사용자 정의 공급자를 사용하는 경우 검증자 API 키
          
          [env: VERIFIER_API_KEY=]

      --verifier-url <VERIFIER_URL>
          사용자 정의 공급자를 사용하는 경우 검증자 URL
          
          [env: VERIFIER_URL=]

Display options:
      --color <COLOR>
          로그 메시지의 색상

          가능한 값:
          - auto:   색상 출력 사용 여부를 지능적으로 추측 (기본값)
          - always: 색상 출력 강제
          - never:  색상 출력 강제 비활성화

      --json
          로그 메시지를 JSON으로 포맷팅

      --md
          로그 메시지를 Markdown으로 포맷팅

  -q, --quiet
          로그 메시지를 인쇄하지 않음

  -v, --verbosity...
          로그 메시지의 상세 수준.
          
          상세 수준을 높이려면 여러 번 전달 (예: -v, -vv, -vvv).
          
          문맥에 따라 상세 수준은 다른 의미를 가짐.
          
          예를 들어, EVM의 상세 수준은 다음과 같음:
          - 2 (-vv): 모든 테스트에 대한 로그 인쇄.
          - 3 (-vvv): 실패한 테스트에 대한 실행 트레이스 인쇄.
          - 4 (-vvvv): 모든 테스트에 대한 실행 트레이스 및 실패한 테스트에 대한 설정(setup) 트레이스 인쇄.
          - 5 (-vvvvv): 스토리지 변경 및 줄 번호가 포함된 백트레이스를 포함하여
            모든 테스트에 대한 실행 및 설정 트레이스 인쇄.
```
