## forge test

프로젝트의 테스트를 실행합니다.

```bash
$ forge test --help
```

```txt
Usage: forge test [OPTIONS] [PATH]

Options:
  -h, --help
          도움말 인쇄 ('-h'로 요약 보기)

Display options:
  -v, --verbosity...
          로그 메시지의 상세 수준.
          
          상세 수준을 높이려면 여러 번 전달 (예: -v, -vv, -vvv).
          
          문맥에 따라 상세 수준은 다른 의미를 가짐.
          
          예를 들어, EVM의 상세 수준은 다음과 같음:
          - 2 (-vv): 모든 테스트에 대한 로그 인쇄.
          - 3 (-vvv): 실패한 테스트에 대한 실행 트레이스 인쇄.
          - 4 (-vvvv): 모든 테스트에 대한 실행 트레이스 및 실패한 테스트에 대한 설정(setup) 트레이스 인쇄.
          - 5 (-vvvvv): 스토리지 변경 및 줄 번호가 포함된 백트레이스를 포함하여
            모든 테스트에 대한 실행 및 설정 트레이스 인쇄.

  -q, --quiet
          로그 메시지를 인쇄하지 않음

      --json
          로그 메시지를 JSON으로 포맷팅

      --md
          로그 메시지를 Markdown으로 포맷팅

      --color <COLOR>
          로그 메시지의 색상

          가능한 값:
          - auto:   색상 출력 사용 여부를 지능적으로 추측 (기본값)
          - always: 색상 출력 강제
          - never:  색상 출력 강제 비활성화

  -s, --suppress-successful-traces
          성공한 테스트 트레이스를 억제하고 실패에 대한 트레이스만 표시
          
          [env: FORGE_SUPPRESS_SUCCESSFUL_TRACES=]

      --junit
          테스트 결과를 JUnit XML 보고서로 출력

  -l, --list
          테스트를 실행하는 대신 목록 표시

      --show-progress
          테스트 실행 진행 상황 표시

      --summary
          테스트 요약 테이블 인쇄

      --detailed
          상세 테스트 요약 테이블 인쇄

      --disable-labels
          트레이스에서 레이블 비활성화

Test options:
  -j, --threads <THREADS>
          사용할 스레드 수. 0으로 지정하면 기본값은 논리 코어 수
          
          [aliases: --jobs]

      --debug
          단일 테스트를 디버거에서 실행.
          
          테스트 결과와 관계없이 일치하는 테스트가 디버거에서 열립니다.
          
          일치하는 테스트가 퍼즈 테스트인 경우 첫 번째 실패 케이스에서
          디버거가 열립니다. 퍼즈 테스트가 실패하지 않으면 마지막 퍼즈
          케이스에서 디버거가 열립니다.

      --flamegraph
          단일 테스트에 대한 플레임그래프(flamegraph) 생성. `--decode-internal`을 암시.
          
          플레임그래프는 스마트 컨트랙트 내의 어떤 함수나 작업이 전체적으로 가장
          많은 가스를 소비하는지 정렬된 방식으로 시각화하는 데 사용됩니다.

      --flamechart
          단일 테스트에 대한 플레임차트(flamechart) 생성. `--decode-internal`을 암시.
          
          플레임차트는 시간 경과에 따른 가스 사용량을 보여주며, 각 함수가
          언제 호출되는지(실행 순서)와 타임라인의 각 지점에서 얼마나 많은
          가스를 소비하는지 보여줍니다.

      --decode-internal
          트레이스에서 내부 함수 식별.
          
          내부 함수를 트레이스하고 스택 매개변수를 디코딩합니다.
          
          메모리에 저장된 매개변수(예: bytes 또는 배열)는 성능상의 이유로
          `--debug`와 유사하게 단일 함수가 일치할 때만 현재 디코딩됩니다.

      --dump <PATH>
          모든 디버거 단계를 파일로 덤프

      --gas-report
          가스 보고서 인쇄
          
          [env: FORGE_GAS_REPORT=]

      --gas-snapshot-check <GAS_SNAPSHOT_CHECK>
          이전 실행과 가스 스냅샷 비교 확인
          
          [env: FORGE_SNAPSHOT_CHECK=]
          [possible values: true, false]

      --gas-snapshot-emit <GAS_SNAPSHOT_EMIT>
          가스 스냅샷 결과 기록 활성화/비활성화
          
          [env: FORGE_SNAPSHOT_EMIT=]
          [possible values: true, false]

      --allow-failure
          테스트가 실패하더라도 코드 0으로 종료
          
          [env: FORGE_ALLOW_FAILURE=]

      --trace-depth <TRACE_DEPTH>
          트레이스의 깊이 정의

      --fail-fast
          첫 번째 실패 후 테스트 실행 중지

      --etherscan-api-key <KEY>
          Etherscan (또는 동등한) API 키
          
          [env: ETHERSCAN_API_KEY=]

      --fuzz-seed <FUZZ_SEED>
          퍼즈 실행 중에 무작위성을 생성하는 데 사용되는 시드 설정

      --fuzz-runs <RUNS>
          [env: FOUNDRY_FUZZ_RUNS=]

      --fuzz-timeout <TIMEOUT>
          각 퍼즈 실행에 대한 타임아웃(초)
          
          [env: FOUNDRY_FUZZ_TIMEOUT=]

      --fuzz-input-file <FUZZ_INPUT_FILE>
          퍼즈 실패를 다시 실행할 파일

      --rerun
          마지막 실행에서 기록된 테스트 실패를 다시 실행. 기록된 실패가 없으면
          일반 테스트 실행 수행

  [PATH]
          테스트하려는 컨트랙트 파일. --match-path의 단축키입니다.

Test filtering:
      --match-test <REGEX>
          지정된 정규식 패턴과 일치하는 테스트 함수만 실행
          
          [aliases: --mt]

      --no-match-test <REGEX>
          지정된 정규식 패턴과 일치하지 않는 테스트 함수만 실행
          
          [aliases: --nmt]

      --match-contract <REGEX>
          지정된 정규식 패턴과 일치하는 컨트랙트의 테스트만 실행
          
          [aliases: --mc]

      --no-match-contract <REGEX>
          지정된 정규식 패턴과 일치하지 않는 컨트랙트의 테스트만 실행
          
          [aliases: --nmc]

      --match-path <GLOB>
          지정된 글로브(glob) 패턴과 일치하는 소스 파일의 테스트만 실행
          
          [aliases: --mp]

      --no-match-path <GLOB>
          지정된 글로브(glob) 패턴과 일치하지 않는 소스 파일의 테스트만 실행
          
          [aliases: --nmp]

      --no-match-coverage <REGEX>
          지정된 정규식 패턴과 일치하지 않는 파일에 대한 커버리지 만 표시
          
          [aliases: --nmco]

EVM options:
  -f, --fork-url <URL>
          빈 상태에서 시작하는 대신 원격 엔드포인트에서 상태 가져오기.
          
          특정 블록 번호에서 상태를 가져오려면 --fork-block-number 참조.
          
          [aliases: --rpc-url]

      --fork-block-number <BLOCK>
          원격 엔드포인트에서 특정 블록 번호의 상태 가져오기.
          
          --fork-url 참조.

      --fork-retries <RETRIES>
          재시도 횟수.
          
          --fork-url 참조.

      --fork-retry-backoff <BACKOFF>
          오류 발생 시 초기 재시도 백오프(backoff).
          
          --fork-url 참조.

      --no-storage-caching
          RPC 캐싱 사용을 명시적으로 비활성화.
          
          모든 스토리지 슬롯을 엔드포인트에서 완전히 읽음.
          
          이 플래그는 프로젝트의 구성 파일을 덮어씀.
          
          --fork-url 참조.

      --initial-balance <BALANCE>
          배포된 테스트 컨트랙트의 초기 잔액

      --sender <ADDRESS>
          테스트/스크립트를 실행할 주소

      --ffi
          FFI 치트코드 활성화

      --always-use-create-2-factory
          테스트 및 비-브로드캐스팅 스크립트를 포함한 모든 경우에 create 2
          팩토리 사용

      --create2-deployer <ADDRESS>
          사용할 CREATE2 배포자 주소. 구성(config)에 있는 주소를 덮어씀

Fork config:

          [https://docs.alchemy.com/reference/compute-units#what-are-cups-compute-units-per-second](https://docs.alchemy.com/reference/compute-units#what-are-cups-compute-units-per-second) 참조

      --no-rpc-rate-limit
          이 노드 공급자에 대한 속도 제한 비활성화.
          
          --fork-url 및
          [https://docs.alchemy.com/reference/compute-units#what-are-cups-compute-units-per-second](https://docs.alchemy.com/reference/compute-units#what-are-cups-compute-units-per-second) 참조
          
          [aliases: --no-rate-limit]

Executor environment config:
      --code-size-limit <CODE_SIZE>
          EIP-170: 바이트 단위의 컨트랙트 코드 크기 제한. 테스트 때문에
          늘리는 데 유용함. 기본값은 0x6000 (~25kb)

      --chain <CHAIN>
          체인 이름 또는 EIP-155 체인 ID
          
          [aliases: --chain-id]

      --gas-price <GAS_PRICE>
          가스 가격

      --block-base-fee-per-gas <FEE>
          블록의 기본 수수료(base fee)
          
          [aliases: --base-fee]

      --tx-origin <ADDRESS>
          트랜잭션 원점(origin)

      --block-coinbase <ADDRESS>
          블록의 코인베이스

      --block-timestamp <TIMESTAMP>
          블록의 타임스탬프

      --block-number <BLOCK>
          블록 번호

      --block-difficulty <DIFFICULTY>
          블록 난이도

      --block-prevrandao <PREVRANDAO>
          블록 prevrandao 값. 참고: 병합(merge) 전에는 이 필드가 mix_hash였음

      --block-gas-limit <BLOCK_GAS_LIMIT>
          블록 가스 제한
          
          [aliases: --gas-limit]

      --memory-limit <MEMORY_LIMIT>
          바이트 단위의 EVM 실행 당 메모리 제한. 이 제한을 초과하면
          `MemoryLimitOOG` 결과가 발생함.
          
          기본값은 128MiB.

      --disable-block-gas-limit
          블록 가스 제한 검사 비활성화 여부
          
          [aliases: --no-block-gas-limit, --no-gas-limit]

      --enable-tx-gas-limit
          Osaka (EIP-7825)에 의해 부과된 트랜잭션 가스 제한 검사 활성화 여부
          
          [aliases: --tx-gas-limit]

      --isolate
          호출 격리 활성화 여부. 격리 모드에서는 모든 최상위 호출이 별도의
          EVM 컨텍스트에서 별도의 트랜잭션으로 실행되어 더 정확한 가스 회계
          및 트랜잭션 상태 변경을 가능하게 함

Cache options:
      --force
          캐시 및 아티팩트 폴더를 지우고 재컴파일

Build options:
      --no-cache
          캐시 비활성화

      --dynamic-test-linking
          동적 테스트 링킹 활성화

      --skip <SKIP>...
          이름에 주어진 필터가 포함된 파일 빌드 건너뛰기.
          
          `test`와 `script`는 `.t.sol`과 `.s.sol`의 별칭입니다.

Linker options:
      --libraries <LIBRARIES>
          사전 링크된 라이브러리 설정
          
          [env: DAPP_LIBRARIES=]

Compiler options:
      --ignored-error-codes <ERROR_CODES>
          오류 코드로 solc 경고 무시

  -D, --deny <LEVEL>
          지정된 진단 수준에서 컴파일러 오류가 트리거됩니다.
          
          사용 중단된 `--deny-warnings` 플래그를 대체합니다.
          
          가능한 값: - `never`: 어떤 진단도 오류로 처리하지 않음. -
          `warnings`: 경고를 오류로 처리. - `notes`: 경고와 노트(notes)
          모두 오류로 처리.

          가능한 값:
          - never:    항상 0 코드로 종료
          - warnings: 경고가 발견되면 0이 아닌 코드로 종료
          - notes:    노트나 경고가 발견되면 0이 아닌 코드로 종료

      --no-auto-detect
          `solc` 버전을 자동 감지하지 않음

      --use <SOLC_VERSION>
          빌드할 solc 버전을 지정하거나 로컬 solc 경로 지정.
          
          유효한 값은 `x.y.z`, `solc:x.y.z` 또는 `path/to/solc` 형식입니다.

      --offline
          네트워크에 액세스하지 않음.
          
          누락된 solc 버전은 설치되지 않음.

      --via-ir
          Yul 중간 표현(intermediate representation) 컴파일 파이프라인 사용

      --use-literal-content
          컴파일 시 URL 대신 리터럴 콘텐츠만 사용하도록 변경

      --no-metadata
          바이트코드에 메타데이터를 추가하지 않음.
          
          `bytecode_hash`를 `none`으로, `cbor_metadata`를 `false`로
          설정하는 것과 동일함.

      --ast
          컴파일러 출력에 AST를 JSON으로 포함

      --evm-version <VERSION>
          대상 EVM 버전

      --optimize [<OPTIMIZE>]
          Solidity 최적화기(optimizer) 활성화
          
          [possible values: true, false]

      --optimizer-runs <RUNS>
          실행 횟수(runs)는 배포된 코드의 각 오퍼코드가 컨트랙트 수명 동안
          대략 얼마나 자주 실행될지를 지정합니다. 이는 코드 크기(배포 비용)와
          코드 실행 비용(배포 후 비용) 간의 균형 매개변수입니다. `optimizer_runs`
          매개변수가 `1`이면 짧지만 비싼 코드를 생성합니다. 반면, 더 큰
          `optimizer_runs` 매개변수는 더 길지만 가스 효율적인 코드를 생성합니다.

      --extra-output <SELECTOR>...
          컨트랙트 아티팩트에 포함할 추가 출력.
          
          예제 키: evm.assembly, ewasm, ir, irOptimized, metadata
          
          전체 설명은 다음을 참조하세요:
          [https://docs.soliditylang.org/en/v0.8.13/using-the-compiler.html#input-description](https://docs.soliditylang.org/en/v0.8.13/using-the-compiler.html#input-description)

      --extra-output-files <SELECTOR>...
          별도 파일에 기록할 추가 출력.
          
          유효한 값: metadata, ir, irOptimized, ewasm, evm.assembly

Project options:

          리버트 문자열 구성.

          문자열) 및 "verboseDebug"

      --build-info
          빌드 정보 파일 생성

      --build-info-path <PATH>
          빌드 정보 파일이 기록될 디렉토리의 출력 경로

      --root <PATH>
          프로젝트의 루트 경로.
          
          기본적으로 Git 저장소의 루트(있는 경우) 또는 현재 작업 디렉토리입니다.

  -C, --contracts <PATH>
          컨트랙트 소스 디렉토리

  -R, --remappings <REMAPPINGS>
          프로젝트의 리매핑

      --remappings-env <ENV>
          환경(변수)에서 가져온 프로젝트 리매핑

      --cache-path <PATH>
          컴파일러 캐시 경로

      --lib-paths <PATH>
          라이브러리 폴더 경로

      --hardhat
          Hardhat 스타일 프로젝트 레이아웃 사용.
          
          다음과 동일함: `--contracts contracts --lib-paths node_modules`.
          
          [aliases: --hh]

      --config-path <FILE>
          구성(config) 파일 경로

Watch options:
  -w, --watch [<PATH>...]
          변경 사항에 대해 주어진 파일 또는 디렉토리 감시.
          
          경로가 제공되지 않으면 프로젝트의 소스 및 테스트 디렉토리가
          감시됩니다.

      --no-restart
          명령이 아직 실행 중일 때 재시작하지 않음

      --run-all
          변경이 발생했을 때 명시적으로 모든 테스트 다시 실행.
          
          기본적으로 마지막으로 수정된 테스트 파일의 테스트만 실행됩니다.

      --rerun-failed
          변경이 발생했을 때 이전에 실패한 테스트만 먼저 다시 실행.
          
          이전에 실패한 모든 테스트가 통과하면 전체 테스트 스위트가 자동으로
          실행됩니다. 이는 실패에 대한 빠른 피드백을 원하는 TDD 워크플로에
          특히 유용합니다.

      --watch-delay <DELAY>
          파일 업데이트 디바운스 지연.
          
          지연 시간 동안 들어오는 변경 이벤트가 누적되며 지연이 지나면
          조치가 취해집니다. 이는 명령이 시작된다는 의미는 아닙니다:
          --no-restart가 주어지고 명령이 이미 실행 중인 경우 조치의 결과는
          아무것도 하지 않는 것입니다.
          
          기본값은 50ms입니다. 기본적으로 십진수 초(seconds)로 파싱되지만,
          `ms` 접미사와 함께 정수를 사용하는 것이 더 편리할 수 있습니다.
          
          --poll 모드를 사용할 때는 더 긴 기간을 원하거나 디스크 I/O 과부하
          위험이 있습니다.
```
