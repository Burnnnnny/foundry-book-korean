## chisel

빠르고 실용적이며 상세한 Solidity REPL

```bash
$ chisel --help
```

```txt
Usage: chisel [OPTIONS] [COMMAND]

Commands:
  list         모든 캐시된 세션 나열
  load         캐시된 세션 로드
  view         캐시된 세션의 소스 보기
  clear-cache  캐시 디렉토리에서 모든 캐시된 chisel 세션 지우기
  eval         REPL에 들어가지 않고 명령어 간단 평가
  help         이 메시지 또는 주어진 하위 명령의 도움말 인쇄

Options:
  -j, --threads <THREADS>
          사용할 스레드 수. 0으로 지정하면 기본값은 논리 코어 수
          
          [aliases: --jobs]

  -h, --help
          도움말 인쇄 ('-h'로 요약 보기)

  -V, --version
          버전 인쇄

Display options:
  -v, --verbosity...
          로그 메시지의 상세 수준.
          
          상세 수준을 높이려면 여러 번 전달 (예: -v, -vv, -vvv).
          
          문맥에 따라 상세 수준은 다른 의미를 가짐.
          
          예를 들어, EVM의 상세 수준은 다음과 같음:
          - 2 (-vv): 모든 테스트에 대한 로그 인쇄.
          - 3 (-vvv): 실패한 테스트에 대한 실행 트레이스 인쇄.
          - 4 (-vvvv): 모든 테스트에 대한 실행 트레이스 및 실패한 테스트에 대한 설정(setup) 트레이스 인쇄.
          - 5 (-vvvvv): 스토리지 변경 및 줄 번호가 포함된 백트레이스를 포함하여
            모든 테스트에 대한 실행 및 설정 트레이스 인쇄.

  -q, --quiet
          로그 메시지를 인쇄하지 않음

      --json
          로그 메시지를 JSON으로 포맷팅

      --md
          로그 메시지를 Markdown으로 포맷팅

      --color <COLOR>
          로그 메시지의 색상

          Possible values:
          - auto:   색상 출력 사용 여부를 지능적으로 추측 (기본값)
          - always: 색상 출력 강제
          - never:  색상 출력 강제 비활성화

REPL options:
      --prelude <PRELUDE>
          가져올 Solidity 파일이 포함된 디렉토리 경로 또는 단일 Solidity 파일 경로.
          
          이 파일들은 REPL의 최상위 레벨 이전에 평가되므로 서곡(prelude) 기능을 함

      --no-vm
          기본 `Vm` 가져오기 비활성화.
          
          Solc 버전이 0.6.2 미만인 경우 기본적으로 가져오기가 비활성화됨.

      --ir-minimum
          최소 최적화로 viaIR 활성화
          
          이것은 대부분의 "stack too deep" 오류를 수정하면서 비교적 정확한 소스 맵을 결과로 냄.

Cache options:
      --force
          캐시 및 아티팩트 폴더를 지우고 다시 컴파일

Build options:
      --no-cache
          캐시 비활성화

      --dynamic-test-linking
          동적 테스트 링킹 활성화

      --skip <SKIP>...
          이름에 주어진 필터가 포함된 파일 빌드 건너뛰기.
          
          `test`와 `script`는 `.t.sol`과 `.s.sol`의 별칭임.

Linker options:
      --libraries <LIBRARIES>
          사전 링크된 라이브러리 설정
          
          [env: DAPP_LIBRARIES=]

Compiler options:
      --ignored-error-codes <ERROR_CODES>
          오류 코드로 solc 경고 무시

  -D, --deny <LEVEL>
          지정된 진단 수준에서 컴파일러 오류가 트리거됨.
          
          더 이상 사용되지 않는 `--deny-warnings` 플래그를 대체함.
          
          가능한 값: - `never`: 어떤 진단도 오류로 처리하지 않음. -
          `warnings`: 경고를 오류로 처리. - `notes`: 경고와 노트 모두를 오류로 처리.
          
          Possible values:
          - never:    항상 0 코드로 종료
          - warnings: 경고가 발견되면 0이 아닌 코드로 종료
          - notes:    노트나 경고가 발견되면 0이 아닌 코드로 종료

      --no-auto-detect
          `solc` 버전을 자동 감지하지 않음

      --use <SOLC_VERSION>
          빌드할 solc 버전을 지정하거나 로컬 solc 경로를 지정.
          
          유효한 값은 `x.y.z`, `solc:x.y.z` 또는 `path/to/solc` 형식임.

      --offline
          네트워크에 액세스하지 않음.
          
          누락된 solc 버전이 설치되지 않음.

      --via-ir
          Yul 중간 표현 컴파일 파이프라인 사용

      --use-literal-content
          컴파일 시 URL 대신 리터럴 내용만 사용하도록 변경

      --no-metadata
          바이트코드에 메타데이터를 추가하지 않음.
          
          이것은 `bytecode_hash`를 `none`으로 설정하고 `cbor_metadata`를 `false`로 설정하는 것과 동일함.

      --ast
          컴파일러 출력에 AST를 JSON으로 포함

      --evm-version <VERSION>
          대상 EVM 버전

      --optimize [<OPTIMIZE>]
          Solidity 최적화 활성화
          
          [possible values: true, false]

      --optimizer-runs <RUNS>
          실행 횟수는 컨트랙트 수명 동안 배포된 코드의 각 연산 코드가 실행될 대략적인 빈도를 지정함.
          이는 코드 크기(배포 비용)와 코드 실행 비용(배포 후 비용) 간의 절충 매개변수임을 의미함.
          `optimizer_runs` 매개변수가 `1`이면 짧지만 비싼 코드가 생성됨. 반면, 더 큰 `optimizer_runs` 매개변수는
          더 길지만 가스 효율적인 코드를 생성함

      --extra-output <SELECTOR>...
          컨트랙트의 아티팩트에 포함할 추가 출력.
          
          Example keys: evm.assembly, ewasm, ir, irOptimized, metadata
          
          전체 설명은 다음 참조
          [https://docs.soliditylang.org/en/v0.8.13/using-the-compiler.html#input-description](https://docs.soliditylang.org/en/v0.8.13/using-the-compiler.html#input-description)

      --extra-output-files <SELECTOR>...
          별도의 파일에 기록할 추가 출력.
          
          Valid values: metadata, ir, irOptimized, ewasm, evm.assembly

Project options:
  -o, --out <PATH>
          컨트랙트 아티팩트 폴더 경로

      --revert-strings <REVERT>
          리버트 문자열 구성.
          
          가능한 값은 "default", "strip" (제거), "debug" (Solidity 생성 리버트 문자열), "verboseDebug"

      --build-info
          빌드 정보 파일 생성

      --build-info-path <PATH>
          빌드 정보 파일이 기록될 디렉토리의 출력 경로

      --root <PATH>
          프로젝트의 루트 경로.
          
          기본적으로 Git 저장소의 루트(있는 경우) 또는 현재 작업 디렉토리.

  -C, --contracts <PATH>
          컨트랙트 소스 디렉토리

  -R, --remappings <REMAPPINGS>
          프로젝트의 리매핑

      --remappings-env <ENV>
          환경에서의 프로젝트 리매핑

      --cache-path <PATH>
          컴파일러 캐시 경로

      --lib-paths <PATH>
          라이브러리 폴더 경로

      --hardhat
          Hardhat 스타일 프로젝트 레이아웃 사용.
          
          이것은 `--contracts contracts --lib-paths node_modules`를 사용하는 것과 동일함.
          
          [aliases: --hh]

      --config-path <FILE>
          구성 파일 경로

EVM options:
  -f, --fork-url <URL>
          빈 상태에서 시작하는 대신 원격 엔드포인트를 통해 상태를 가져옴.
          
          특정 블록 번호에서 상태를 가져오려면 --fork-block-number 참조.
          
          [aliases: --rpc-url]

      --fork-block-number <BLOCK>
          원격 엔드포인트를 통해 특정 블록 번호에서 상태를 가져옴.
          
          --fork-url 참조.

      --fork-retries <RETRIES>
          재시도 횟수.
          
          --fork-url 참조.

      --fork-retry-backoff <BACKOFF>
          오류 발생 시 초기 재시도 백오프.
          
          --fork-url 참조.

      --no-storage-caching
          RPC 캐싱 사용을 명시적으로 비활성화.
          
          모든 스토리지 슬롯은 엔드포인트에서 완전히 읽음.
          
          이 플래그는 프로젝트의 구성 파일을 재정의함.
          
          --fork-url 참조.

      --initial-balance <BALANCE>
          배포된 테스트 컨트랙트의 초기 잔액

      --sender <ADDRESS>
          테스트/스크립트를 실행할 주소

      --ffi
          FFI 치트코드 활성화

      --always-use-create-2-factory
          테스트 및 비브로드캐스팅 스크립트를 포함한 모든 경우에 create 2 팩토리 사용

      --create2-deployer <ADDRESS>
          사용할 CREATE2 배포자 주소, 구성에 있는 주소를 재정의함

Fork config:
      --compute-units-per-second <CUPS>
          이 공급자에 대해 가정된 초당 사용 가능한 컴퓨팅 단위 수를 설정
          
          기본값: 330
          
          --fork-url 및
          [https://docs.alchemy.com/reference/compute-units#what-are-cups-compute-units-per-second](https://docs.alchemy.com/reference/compute-units#what-are-cups-compute-units-per-second) 참조

      --no-rpc-rate-limit
          이 노드 공급자에 대한 속도 제한을 비활성화.
          
          --fork-url 및
          [https://docs.alchemy.com/reference/compute-units#what-are-cups-compute-units-per-second](https://docs.alchemy.com/reference/compute-units#what-are-cups-compute-units-per-second) 참조
          
          [aliases: --no-rate-limit]

Executor environment config:
      --code-size-limit <CODE_SIZE>
          EIP-170: 바이트 단위의 컨트랙트 코드 크기 제한. 테스트 때문에 늘리는 데 유용함.
          기본값은 0x6000 (~25kb)

      --chain <CHAIN>
          체인 이름 또는 EIP-155 체인 ID
          
          [aliases: --chain-id]

      --gas-price <GAS_PRICE>
          가스 가격

      --block-base-fee-per-gas <FEE>
          블록의 기본 수수료
          
          [aliases: --base-fee]

      --tx-origin <ADDRESS>
          트랜잭션 원점(origin)

      --block-coinbase <ADDRESS>
          블록의 코인베이스

      --block-timestamp <TIMESTAMP>
          블록의 타임스탬프

      --block-number <BLOCK>
          블록 번호

      --block-difficulty <DIFFICULTY>
          블록 난이도

      --block-prevrandao <PREVRANDAO>
          블록 prevrandao 값. 참고: 병합(merge) 전에는 이 필드가 mix_hash였음

      --block-gas-limit <BLOCK_GAS_LIMIT>
          블록 가스 제한
          
          [aliases: --gas-limit]

      --memory-limit <MEMORY_LIMIT>
          EVM 실행 당 메모리 제한(바이트). 이 제한을 초과하면 `MemoryLimitOOG` 결과가 발생함.
          
          기본값은 128MiB.

      --disable-block-gas-limit
          블록 가스 제한 확인을 비활성화할지 여부
          
          [aliases: --no-block-gas-limit, --no-gas-limit]

      --enable-tx-gas-limit
          Osaka (EIP-7825)에 의해 부과된 tx 가스 제한 확인을 활성화할지 여부
          
          [aliases: --tx-gas-limit]

      --isolate
          호출 격리를 활성화할지 여부. 격리 모드에서는 모든 최상위 호출이 별도의 EVM 컨텍스트에서
          별도의 트랜잭션으로 실행되어, 더 정밀한 가스 계산 및 트랜잭션 상태 변경이 가능함
```
