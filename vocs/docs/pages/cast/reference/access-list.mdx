## cast access-list

트랜잭션에 대한 액세스 목록(access list)을 생성합니다.

```bash
$ cast access-list --help
```

```txt
Usage: cast access-list [OPTIONS] [TO] [SIG] [ARGS]...

Arguments:
  [TO]
          트랜잭션의 수신자

  [SIG]
          호출할 함수의 서명

  [ARGS]...
          호출할 함수의 인수

Options:
  -B, --block <BLOCK>
          쿼리할 블록 높이.
          
          earliest, finalized, safe, latest, 또는 pending 태그도 가능합니다.

  -h, --help
          도움말 인쇄 ('-h'로 요약 보기)

  -j, --threads <THREADS>
          사용할 스레드 수. 0으로 지정하면 기본값은 논리 코어 수
          
          [aliases: --jobs]

Transaction options:
      --gas-limit <GAS_LIMIT>
          트랜잭션의 가스 제한
          
          [env: ETH_GAS_LIMIT=]

      --gas-price <PRICE>
          레거시 트랜잭션의 가스 가격, 또는 EIP1559 트랜잭션의 가스당 최대 수수료.
          wei로 지정하거나 단위 유형이 있는 문자열로 지정합니다.
          
          Examples: 1ether, 10gwei, 0.01ether
          
          [env: ETH_GAS_PRICE=]

      --priority-gas-price <PRICE>
          EIP1559 트랜잭션의 가스당 최대 우선 수수료
          
          [env: ETH_PRIORITY_GAS_PRICE=]

      --value <VALUE>
          트랜잭션에 보낼 이더. wei로 지정하거나 단위 유형이 있는 문자열로 지정합니다.
          
          Examples: 1ether, 10gwei, 0.01ether

      --nonce <NONCE>
          트랜잭션의 논스(Nonce)

      --legacy
          EIP1559 트랜잭션 대신 레거시 트랜잭션을 보냅니다.
          
          EIP1559가 없는 일반적인 네트워크에서는 자동으로 활성화됩니다.

      --blob
          EIP-4844 블롭(blob) 트랜잭션을 보냅니다.

      --blob-gas-price <BLOB_PRICE>
          EIP-4844 블롭 트랜잭션의 가스 가격
          
          [env: ETH_BLOB_GAS_PRICE=]

      --auth <AUTH>
          EIP-7702 인증 목록.
          
          16진수로 인코딩된 서명된 인증 또는 주소일 수 있습니다.

      --access-list [<ACCESS_LIST>]
          EIP-2930 액세스 목록.
          
          JSON으로 인코딩된 액세스 목록을 받거나, 빈 값을 받아 `eth_createAccessList`에 대한
          RPC 호출을 통해 액세스 목록을 생성합니다. 액세스 목록 부분만 검색하려면
          `cast access-list` 명령어를 사용하세요.

Rpc options:
  -r, --rpc-url <URL>
          RPC 엔드포인트. 기본값은 http://localhost:8545
          
          [env: ETH_RPC_URL=]

  -k, --insecure
          안전하지 않은 RPC 연결 허용 (유효하지 않은 HTTPS 인증서 수락).
          
          공급자의 내부 런타임 전송 변형이 HTTP일 때, 이 옵션은 reqwest 클라이언트가
          유효하지 않은 인증서를 수락하도록 구성합니다.

      --flashbots
          Flashbots RPC URL을 빠른 모드로 사용
          ([https://rpc.flashbots.net/fast](https://rpc.flashbots.net/fast)).
          
          이는 모든 등록된 빌더와 트랜잭션을 비공개로 공유합니다.
          
          참조:
          [https://docs.flashbots.net/flashbots-protect/quick-start#faster-transactions](https://docs.flashbots.net/flashbots-protect/quick-start#faster-transactions)

      --jwt-secret <JWT_SECRET>
          RPC 엔드포인트의 JWT 시크릿.
          
          JWT 시크릿은 RPC용 JWT를 생성하는 데 사용됩니다. 예를 들어, 다음을 사용하여
          CL `engine_forkchoiceUpdated` 호출을 시뮬레이션할 수 있습니다:
          
          cast rpc --jwt-secret <JWT_SECRET> engine_forkchoiceUpdatedV2
          '["0x6bb38c26db65749ab6e472080a3d20a2f35776494e72016d1e339593f21c59bc",
          "0x6bb38c26db65749ab6e472080a3d20a2f35776494e72016d1e339593f21c59bc",
          "0x6bb38c26db65749ab6e472080a3d20a2f35776494e72016d1e339593f21c59bc"]'
          
          [env: ETH_RPC_JWT_SECRET=]

      --rpc-timeout <RPC_TIMEOUT>
          RPC 요청의 시간 초과(초).
          
          지정된 시간 초과는 RPC 요청의 기본 시간 초과를 재정의하는 데 사용됩니다.
          
          Default value: 45
          
          [env: ETH_RPC_TIMEOUT=]

      --rpc-headers <RPC_HEADERS>
          RPC 요청에 대한 사용자 정의 헤더 지정
          
          [env: ETH_RPC_HEADERS=]

Wallet options - raw:
  -f, --from <ADDRESS>
          발신자 계정
          
          [env: ETH_FROM=]

  -i, --interactive
          개인 키를 입력할 대화형 프롬프트 열기

      --private-key <RAW_PRIVATE_KEY>
          제공된 개인 키 사용

      --mnemonic <MNEMONIC>
          지정된 경로의 니모닉 파일 또는 니모닉 문구 사용

      --mnemonic-passphrase <PASSPHRASE>
          니모닉에 BIP39 암호 문구 사용

      --mnemonic-derivation-path <PATH>
          지갑 파생 경로.
          
          --mnemonic-path 및 하드웨어 지갑 모두에서 작동합니다.

      --mnemonic-index <INDEX>
          주어진 니모닉 인덱스의 개인 키 사용.
          
          --mnemonic-path와 함께 사용됩니다.
          
          [default: 0]

Wallet options - keystore:
      --keystore <PATH>
          주어진 폴더 또는 파일의 키스토어 사용
          
          [env: ETH_KEYSTORE=]

      --account <ACCOUNT_NAME>
          파일 이름을 사용하여 기본 키스토어 폴더(~/.foundry/keystores)의 키스토어 사용
          
          [env: ETH_KEYSTORE_ACCOUNT=]

      --password <PASSWORD>
          키스토어 비밀번호.
          
          --keystore와 함께 사용됩니다.

      --password-file <PASSWORD_FILE>
          키스토어 비밀번호 파일 경로.
          
          --keystore와 함께 사용됩니다.
          
          [env: ETH_PASSWORD=]

Wallet options - hardware wallet:
  -l, --ledger
          Ledger 하드웨어 지갑 사용

  -t, --trezor
          Trezor 하드웨어 지갑 사용

Wallet options - remote:
      --aws
          AWS Key Management Service 사용.
          
          AWS_KMS_KEY_ID 환경 변수가 설정되어 있는지 확인하세요.

      --gcp
          Google Cloud Key Management Service 사용.
          
          다음 환경 변수가 설정되어 있는지 확인하세요: GCP_PROJECT_ID,
          GCP_LOCATION, GCP_KEY_RING, GCP_KEY_NAME, GCP_KEY_VERSION.
          
          참조: [https://cloud.google.com/kms/docs](https://cloud.google.com/kms/docs)

      --turnkey
          Turnkey 사용.
          
          다음 환경 변수가 설정되어 있는지 확인하세요:
          TURNKEY_API_PRIVATE_KEY, TURNKEY_ORGANIZATION_ID, TURNKEY_ADDRESS.
          
          참조: [https://docs.turnkey.com/getting-started/quickstart](https://docs.turnkey.com/getting-started/quickstart)

Wallet options - browser:
      --browser
          브라우저 지갑 사용

      --browser-port <PORT>
          브라우저 지갑 서버 포트
          
          [default: 9545]

      --browser-disable-open
          지갑 연결을 위해 브라우저를 열지 여부

Display options:
      --color <COLOR>
          로그 메시지의 색상

          가능한 값:
          - auto:   색상 출력 사용 여부를 지능적으로 추측 (기본값)
          - always: 색상 출력 강제
          - never:  색상 출력 강제 비활성화

      --json
          로그 메시지를 JSON으로 포맷팅

      --md
          로그 메시지를 Markdown으로 포맷팅

  -q, --quiet
          로그 메시지를 인쇄하지 않음

  -v, --verbosity...
          로그 메시지의 상세 수준.
          
          상세 수준을 높이려면 여러 번 전달 (예: -v, -vv, -vvv).
          
          문맥에 따라 상세 수준은 다른 의미를 가짐.
          
          예를 들어, EVM의 상세 수준은 다음과 같음:
          - 2 (-vv): 모든 테스트에 대한 로그 인쇄.
          - 3 (-vvv): 실패한 테스트에 대한 실행 트레이스 인쇄.
          - 4 (-vvvv): 모든 테스트에 대한 실행 트레이스 및 실패한 테스트에 대한 설정(setup) 트레이스 인쇄.
          - 5 (-vvvvv): 스토리지 변경 및 줄 번호가 포함된 백트레이스를 포함하여
            모든 테스트에 대한 실행 및 설정 트레이스 인쇄.
```
