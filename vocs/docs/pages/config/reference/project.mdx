## Project

일반적인 프로젝트 관련 구성입니다.

##### `extends`

- Type: string 또는 object
- Default: 없음
- Environment: N/A

기본 TOML 파일에서 상속하기 위한 구성입니다. 다음 중 하나로 지정할 수 있습니다:
- 문자열 경로: `extends = "./base-config.toml"`
- 경로와 전략이 포함된 객체: `extends = { path = "./base-config.toml", strategy = "replace-arrays" }`

경로는 `foundry.toml` 파일의 위치에 상대적이어야 합니다.

병합 전략, 우선 순위 규칙 및 예제에 대한 자세한 정보는 [구성 상속(Configuration Inheritance)](/guides/project-setup/project-layout#configuration-inheritance) 섹션을 참조하세요.

##### `src`

- Type: string
- Default: src
- Environment: `FOUNDRY_SRC` 또는 `DAPP_SRC`

프로젝트 루트에 상대적인 컨트랙트 소스 경로입니다.

##### `test`

- Type: string
- Default: test
- Environment: `FOUNDRY_TEST` 또는 `DAPP_TEST`

프로젝트 루트에 상대적인 테스트 컨트랙트 소스 경로입니다.

##### `script`

- Type: string
- Default: script
- Environment: `FOUNDRY_SCRIPT` 또는 `DAPP_SCRIPT`

프로젝트 루트에 상대적인 스크립트 컨트랙트 소스 경로입니다.

##### `out`

- Type: string
- Default: out
- Environment: `FOUNDRY_OUT` 또는 `DAPP_OUT`

프로젝트 루트에 상대적인 컨트랙트 아티팩트를 저장할 경로입니다.

##### `libs`

- Type: array of strings (paths)
- Default: lib
- Environment: `FOUNDRY_LIBS` 또는 `DAPP_LIBS`

프로젝트 루트에 상대적인 라이브러리를 포함하는 경로들의 배열입니다.

##### `cache`

- Type: boolean
- Default: true
- Environment: `FOUNDRY_CACHE` 또는 `DAPP_CACHE`

캐싱 활성화 여부입니다. 활성화되면 소스, 테스트 및 종속성의 컴파일 결과가 `cache`에 캐시됩니다.

##### `cache_path`

- Type: string
- Default: cache
- Environment: `FOUNDRY_CACHE_PATH` 또는 `DAPP_CACHE_PATH`

프로젝트 루트에 상대적인 캐시 경로입니다.

##### `broadcast`

- Type: string
- Default: broadcast

프로젝트 루트에 상대적인 브로드캐스트 트랜잭션 로그 경로입니다.

##### `force`

- Type: boolean
- Default: false
- Environment: `FOUNDRY_FORCE` 또는 `DAPP_FORCE`

캐시를 버리고 클린 빌드를 수행할지 여부입니다.
